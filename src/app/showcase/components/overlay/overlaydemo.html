Overlay API

这个API允许从PrimeNGConfig控制覆盖层组件。通过这种方式，应用程序中的所有覆盖组件可以具有相同的行为.

选项
Mode

它有两个有效值; overlay 和 modal. 在 overlay 模式下, 会像叠加面板或下拉面板一样打开一个容器元素. 在 modal 模式下, 容器元素的行为类似于popup. 这种行为类似于dialog对话框组件.

import { PrimeNGConfig, OverlayOptions } from 'primeng/api'; @Component({ selector: 'app-root', templateUrl: './app.component.html' }) export class AppComponent implements OnInit { constructor(private primengConfig: PrimeNGConfig) {} ngOnInit() { this.primengConfig.overlayOptions: OverlayOptions = { mode: 'modal' }; } }
Responsive

它是用来根据给定的media或breakpoint确定它应该以何种模式出现的选项.

import { PrimeNGConfig, OverlayOptions, ResponsiveOverlayDirectionType } from 'primeng/api'; const responsiveOptions: ResponsiveOverlayOptions = { // style?: any; // 给定断点或媒体查询中的组件样式 // styleClass?: string; // 给定断点或媒体查询中的组件样式类 // contentStyle?: any; // 给定断点或媒体查询中的内容样式 // contentStyleClass?: string; // 给定断点或媒体查询中的内容样式类 // breakpoint?: string; // 模态模式下显示组件所需的断点。Exp: ` 640px `， ` 10rem `等。. // media?: string; // 模态模式下显示组件所需的媒体查询. Exp: '@media screen and (max-width: 640px)', '@media screen and (min-width: 640px) and (max-width: 900px)' 等. // direction?: ResponsiveOverlayDirectionType; // 组件在模态模式下显示的方向. } this.primengConfig.overlayOptions: OverlayOptions = { responsive: responsiveOptions };

direction属性的有效值:

  • center (默认值)
  • top
  • top-start
  • top-end
  • bottom
  • bottom-start
  • bottom-end
  • left
  • left-start
  • left-end
  • right
  • right-start
  • right-end
AppendTo

使用这个选项，覆盖层可以挂载到它的location, body 或DOM元素实例中.

import { PrimeNGConfig, OverlayOptions } from 'primeng/api'; this.primengConfig.overlayOptions: OverlayOptions = { appendTo: 'body' };
Target

target用于检测那些相对覆盖层被定位的元素。有效值是;

  • @prev (默认值)
  • @next
  • @parent
  • @grandparent
  • 使用 CSS 选择器
  • 使用 () => HTMLElement
Style and StyleClass

style和styleClass用于定义将添加到所有覆盖层组件的样式。此外，它可以用于contentStyle和contentStyleClass类

AutoZIndex

autoZIndex决定是否自动管理分层。其默认值为`false`.

BaseZIndex

baseZIndex 是在分层中使用的基础zIndex值。默认值为0.

ShowTransitionOptions and HideTransitionOptions

显示或隐藏动画的过渡选项。showTransitionOptions的默认值是`.12s cubic-bezier(0,0,0.2, 1)`并且hideTransitionOptions的默认值是`.1s linear`.

事件
import { PrimeNGConfig, OverlayOptions, OverlayOnBeforeShowEvent, OverlayOnShowEvent, OverlayOnBeforeHideEvent, OverlayOnHideEvent } from 'primeng/api'; import { AnimationEvent } from '@angular/animations'; this.primengConfig.overlayOptions: OverlayOptions = { onBeforeShow: (event?: OverlayOnBeforeShowEvent) => {}; // 在覆盖层显示之前调用的回调函数. onShow: (event?: OverlayOnShowEvent) => {}; // 当覆盖层显示时调用的回调函数. onBeforeHide: (event?: OverlayOnBeforeHideEvent) => {}; // 在覆盖层被隐藏之前调用的回调函数. onHide: (event?: OverlayOnHideEvent) => {}; // 当覆盖层被隐藏时调用的回调函数. onAnimationStart: (event?: AnimationEvent) => {}; // 在动画开始时调用的回调函数. onAnimationDone: (event?: AnimationEvent) => {}; // 当动画完成时调用的回调函数. };
组件

p-overlay 是一个在Overlay窗口中显示内容的容器。上面提到的所有选项都可以用作此组件的props.

Content
导入
import { OverlayModule } from 'primeng/overlay';
准备开始
<button (click)="toggle()">显示 Overlay</button> <p-overlay [(visible)]="overlayVisible" [responsive]="{ breakpoint: '640px', direction: 'bottom', contentStyleClass: 'h-20rem' }" contentStyleClass="p-4 surface-overlay shadow-2 border-round"> Content </p-overlay> import { Component } from '@angular/core'; @Component({ templateUrl: './overlaydemo.html' }) export class OverlayDemo { overlayVisible: boolean = false; toggle() { this.overlayVisible = !this.overlayVisible; } }
模板

可以使用content模板自定义内容.

<button (click)="toggle()">Show Overlay</button> <p-overlay [(visible)]="overlayVisible" [responsive]="{ breakpoint: '640px', direction: 'bottom', contentStyleClass: 'h-20rem' }" contentStyleClass="p-4 surface-overlay shadow-2 border-round"> <ng-template pTemplate="content" let-option> Content - {{option.mode}} </ng-template> </p-overlay>
